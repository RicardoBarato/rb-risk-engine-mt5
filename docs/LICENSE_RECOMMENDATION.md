# License Recommendation

No license file was added in this public hardening pass.

## Option A: No license

- More control.
- People can view the code, but they do not receive broad explicit permission to reuse it.
- Recommended for now while public/private boundaries are still being finalized.

## Option B: MIT

- Very open and common for portfolio projects.
- Allows broad reuse by third parties.
- Not ideal if you want to restrict commercial reuse of the public framework.

## Option C: Apache 2.0

- Open-source license with patent language.
- Useful for larger software projects.
- Probably more than this repository needs right now.

## Option D: Custom educational license

- Could allow learning and review while restricting commercial use.
- Requires careful wording and possibly legal review.
- May be a better future option if the public repository remains educational but not fully open for reuse.

## Current recommendation

Keep the repository without a license until the public/private split is complete.
